What is Git? How to use Git and Git commands?

Published on July 22, 2021 by Khushboo Dhiman on Web Technologies

Git is a version control system designed to handle projects. The project sizes can vary from small to large where programmers need to collaborate. So they can create branches of work they do and then easily merge it with many more powerful features.

Install Git

You can easily get git from https://git-scm.com/book/en/v2/Getting-Started-Installing-Git. Just follow the default settings if you dont want to mess with anything or if you want to explore then nothing can stop you. Go ahead!

Git Bash

Git Bash is the terminal for executing git commands. Bash is an acronym for Bourne Again Shell. A shell is a terminal application used to interface with an operating system through written commands.

Two ways to setup git:

  • Initialize a repository.
  • Clone an existing repository.

    • Setup Git

    Introduce yourself with git. Let it know what to call you and add your email using this :

    git config --global user.name `Your-name`

    git config --global user.email `Your-email`

    Go to folder from File explorer; right click to open Git Bash here. Then to initialize a repository use

    git init

    Stages of files:

    Untracked

    Git has nothing to do with these files. This is the default stage of files.

    Staged

    From untracked when the user adds these files to the git's working tree, it is added to the stage from where the files are ready to commit and then pushed on the remote server.

    UnModified

    Once the file is pushed, its first version will be unmodified until we modify it. Whenever you check the status of git it will show you the stage of all the files whether it is modified, unmodified or else.

    Modified

    Once the file is changed even a single word is changed, then it will be considered as modified and git will show it to you. and this file is now again ready to be staged.

    Git Status

    You will surely use this commands a million times. This command shows the status or stage of files in directory.
    Which files are getting tracked?
    Which are untracked?
    Which are modified?
    Which are staged and which aren't?
    On which branch you are currently working?
    and much more...

    git status

    Adding a file to stage

    git add filename
    or
    git add -A (to add all files in the directory at once)

    To commit i.e. Save

    To start tracking in your project, you first need to make an initial commit

    git commit

    In git bash, with just git commit command it opens up the vim editor where you need to first go to the insert mode by pressing i on your keyboard and then type the message for your commit (As git is for collaborating, so your co-workers should be able to quickly get what you did with your code by checking the commit message) and press Esc to exit then insert mode. Press :wq to quit and save the vim editor.

    But you can escape this if you want by using:

    git commit -m "Your message here"

    Create Blank file

    touch filename

    To recover a file

    If accidently you've made wrong commit in a file and you want changes to revert to one step back, you can use git checkout filename. This will take you one-step back even if you have committed your changes(Here you can take back your words XD))

    git checkout -f (for all files to revert one step back)

    Keep record of versions

    It will list author, commits with commit messages, date and time.

    git log
    git log -p -1 or -2 (how many last logs you want to check)

    git diff command

    This command will tell the difference between the previous version and modified file.
    What's added?
    What's removed?

    If working directory and staged file has no difference it will show nothing.

    git diff (compares working area with staging area)
    git diff -staged (compares staging area with last commit)

    Remove file

    git rm

    git rm removed the working area and staging area. If you want to remove a file from staging area but not from working area, then use: git rm --cached filename

    Ignore file

    If you want to keep some files untracked forever, then make .gitignore file with git bash using touch command and then just list the name of files which you dont want to keep in track

    After that if you try to add or commit, those files won't be tracked. Git will ignore them.

    {These files can be some heavy files in project which aren't directly related to project or log files which are nto needed to be shared with others}

    Branches

    Master is the main branch in git. Here you have a working tree and branches. If you want to make changes to some files and you dont want to create to waste your time to create a copy of project and make changes, Branches can help you a lot. You can easily create your own branch in which you can make changes which will not be reflected towards the master. In this way, you created your own version of the project.

    Create new Branch

    git branch branch-name

    To view all branches of project

    git branch - This shows the current branch in green color

    To jump to any branch

    git checkout branch-name

    To merge 2 branches

    git merge new-branch

    Create a new branch and move to that

    git checkout -b branch-name

    Cloning a project

    git clone url folder-name
