React Learning (Day-3)

Published on August 9, 2021 by Khushboo Dhiman on Web Technologies

React Routers Day.

I think React router is the solution of everything I was looking for.

Key things that I learnt today.

1. Basics of React Router.

2. Installing React Router.

3. Preventing the Broswer from default reload.

4. Style Nav active States.

Tasks I completed today.

1. Implementing the Navbar and viewing different views on Click.


1. Basics of React Router.

React routers are used to implement Client side routing (user navigates through the website without page reloading but URL changes using JavaScript). We can update the URL and change the UI based on that URL and this doesn't send request to the server. So its still a single page application. I was stuck for days because of the navigation and single page item view. But React Router was the solution :D

What is Routing?

Routing is used to provide routes to different pages in react and render components based on that page. We can write the code of doing this by ourselves but we have a package which provides all these features so why? The package is react-router-dom.

2. Installing React Router.

React Routers are not in-built so we have to install them explicitly. Using npm install react-router-dom

To import this we have to use import { Route } from 'react-router-dom' in app.js. And then use Route as a component and pass the path in the path attribute. But to implement this properly, we have to do some more; import { BrowserRouter } from 'react-router-dom' and wrap the App component around it in the main file (inde.js)

Best practice is to store these pages components in a folder "pages" or "screens".

3. Preventing the Broswer from default reload.

import { Links } from 'react-router-dom'; and use Link instead of anchor tags in navigations. This also take the to attribute. This link will stop the default request process of the browser and our app will still remain single page app.

4. Style Nav active States.

Instead of using Link we can use NavLink, everything is same its just that this provides a attribute to set the styling of active class in the navbar which makes our work more easier. So the style of active class given in attribute will be applied on the element if its active. Simple right?
