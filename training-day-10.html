React Learning (Day-2)

Published on August 5, 2021 by Khushboo Dhiman on Web Technologies

React Learning.

Today, I started the next module of React course. Its main focus is - State, Handling Events and Updating UI and working with State.

Key things that I learnt today.

1. Basics of React State.

2. Reacting on a button click.

3. React useState Hook.

3. Update the state of React.


1. Basics of React State.

State is the main reason why React reacts :P (lame joke, but its truth). Without using state, even if change something, some variable or value in our project, react is going to ignore it(Huh, I dont care). But using state we got that power to force react that you need to re-render this component because something has changed.

2. React to a button click.

Event listeners in react start with "on" like onClick, onHover and so on. Now, all event handlers need a function which specifies what needs to be done when this event occurs. You can declare a function in a component function itself.

Here we just need to point to function not call it because if we call it, its going to execute when this function or component renders and this would be so early. So we will just point it here and react will understand that this function needs to be executed only when this event occurs.

Its considered a best pratice to name the event handlers functions ending with the work Handler like clickHandler. So to use React States we need to import it import {useState} from 'react';

3. React useState Hook.

In React, hooks usually start with 'use'. And can only be called inside a component. useState forces react to re-render the component when something has changed. So we need to specify that for which variables or things react has to re-render on change. We call it the initial value and when that initial value changes, re-render.

useState hook returns an object. From which we mostly need two thigns - variable and function-to-update-variable-state. So we can use object destructuring for this.
